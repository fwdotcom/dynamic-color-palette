Godot 4 spatial shader for the [Dynamic Color Palette](https://github.com/fwdotcom/dynamic-color-palette) workflow. Licensed under the **MIT License** — use freely in commercial and open-source Godot projects.

---

## Requirements

- Godot **4.x** (not compatible with Godot 3)
- Textures generated by the [DCP Blender addon](../dynamic_color_palette/README.md)

---

## Installation

1. Copy `dynamic_color_palette.gdshader` into your Godot project (e.g. `res://shaders/`).
2. In Godot, select your mesh node.
3. In the Inspector, assign a new **ShaderMaterial** to the mesh's material slot.
4. Set the ShaderMaterial's **Shader** property to `dynamic_color_palette.gdshader`.

---

## Texture Setup

The shader requires two textures exported from Blender by the DCP addon:

| Uniform | File | Import settings |
|---|---|---|
| `albedo_tex` | `dcp_albedo.png` | **Color → sRGB** (default), Filter: **Nearest**, Repeat: **Disabled** |
| `material_tex` | `dcp_material.png` | **Color → Linear** (non-color data), Filter: **Nearest**, Repeat: **Disabled** |

**Critical import settings:**

- Set `material_tex` to **non-color** (linear). In Godot's import dock: **Compress → Mode: Lossless** and uncheck **Detect 3D** / ensure Color Space is set to Linear. If Godot applies sRGB to this texture, PBR values will be wrong.
- Use **Nearest** filtering on both textures. Bilinear filtering blurs adjacent palette cells into each other, producing incorrect colors at cell edges.
- Disable **Repeat** on both textures to prevent UV wrap-around at the palette border.

Assign the textures to the ShaderMaterial's shader parameters:

```
albedo_tex   → dcp_albedo.png
material_tex → dcp_material.png
```

---

## Shader Parameters

| Parameter | Type | Default | Description |
|---|---|---|---|
| `albedo_tex` | `sampler2D` | — | HSV color palette texture (sRGB) |
| `material_tex` | `sampler2D` | — | PBR data texture (linear). R=Roughness, G=Metallic, B=Emission Strength |
| `emission_scale` | `float` | `1.0` | Global emission multiplier (range 0–16). Scale emission at runtime without modifying textures. Set to `0.0` to disable emission entirely. |

### Channel mapping of `material_tex`

```
R (red)   → Roughness
G (green) → Metallic
B (blue)  → Emission Strength  (multiplied with albedo color × emission_scale)
```

## UV Requirements

The shader reads both textures at the mesh's UV coordinate. UVs must point to the **center of a palette cell** in the correct quadrant:

```
┌─────────────┬─────────────┐
│  Solid      │  Metal      │
│  (top-left) │  (top-right)│
├─────────────┼─────────────┤
│  Emission   │  Info       │
│  (bot-left) │  (bot-right)│
└─────────────┴─────────────┘
```

UV islands pointing outside a quadrant boundary will sample mixed colors from adjacent cells or the info quadrant. Use the DCP addon's **Assign Multicolor Material** operator to place UVs precisely — it calculates exact cell-center coordinates for you.

For the emission quadrant, the cell is subdivided into vertical strips (bottom = weakest, top = strongest). The addon places UVs at the correct vertical position for the chosen strip.

## Usage with Singlecolor Materials

If you used the **Singlecolor** workflow in the Blender addon, no shader setup is needed. Singlecolor materials are standard Principled BSDF materials with flat color and PBR values baked in. Godot imports them automatically via glTF or FBX. Assign this shader only to meshes using the **Multicolor** workflow.

## Runtime Emission Control

The `emission_scale` uniform can be animated or scripted at runtime:

```gdscript
# Example: pulse emission in and out
func _process(delta: float) -> void:
    var mat := $MeshInstance3D.get_active_material(0) as ShaderMaterial
    if mat:
        mat.set_shader_parameter("emission_scale", abs(sin(Time.get_ticks_msec() * 0.001)) * 4.0)
```

This is useful for flickering lights, pulsing effects, or globally toggling emission without modifying the texture.
